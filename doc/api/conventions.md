# API Conventions

## Base

- REST JSON API
- Prefix: /api/v1
- UTF-8
- OpenAPI generated by FastAPI
- Resource names ใช้ plural kebab-case

## Authentication

กลไก final ขึ้นกับ OD-001 แต่ทุก protected endpoint ใช้ dependency เดียวกันและตอบ 401/403 ตามความหมาย

## Resource Examples

    /api/v1/auth
    /api/v1/personnel-imports
    /api/v1/org-units
    /api/v1/question-banks
    /api/v1/questions
    /api/v1/exam-papers
    /api/v1/exam-windows
    /api/v1/exam-sessions
    /api/v1/reports

## HTTP Semantics

| Method | Use |
|---|---|
| GET | Read, no state change |
| POST | Create or command with state transition |
| PUT | Idempotent replacement/upsert such as answer |
| PATCH | Partial update of allowed draft resource |
| DELETE | Avoid for historical data; archive command preferred |

## Status Codes

- 200 successful read/command returning data
- 201 resource created
- 204 successful command without body
- 400 malformed/invalid generic request
- 401 unauthenticated
- 403 authenticated but forbidden
- 404 resource not visible/not found
- 409 state conflict, duplicate or expired workflow
- 422 field validation
- 500 unexpected server error with correlation ID

## Success Envelope

Simple resource responses ไม่ต้องห่อ data โดยไม่มีเหตุผล:

    {
      "id": "...",
      "status": "in_progress"
    }

List response:

    {
      "items": [],
      "page": 1,
      "page_size": 25,
      "total": 0
    }

## Error Envelope

    {
      "error": {
        "code": "EXAM_TIMED_OUT",
        "message": "The examination time has ended.",
        "field_errors": [],
        "correlation_id": "..."
      }
    }

Rules:

- code เสถียรและใช้โดย Frontend
- message อ่านได้แต่ไม่ใช้เป็น programmatic key
- field_errors มี field, code, message
- production ไม่ส่ง stack trace

## Time

- ISO 8601 UTC with Z
- Response exam session ส่ง server_time และ ends_at
- Backend ไม่รับ client timestamp เป็น authority

## IDs

- Expose UUID string
- Choice answer ส่ง selected_choice_id
- ห้ามส่ง national identifier เป็น resource ID

## Pagination and Sorting

- page เริ่ม 1
- page_size มี maximum
- sort fields ต้อง allowlist
- มี deterministic tie-breaker ด้วย ID

## Filtering

- Query parameters ชัดเจน
- Backend บังคับ permission scope เพิ่มจาก filter เสมอ
- Client ส่ง org_unit_id ไม่สามารถขยาย scope ตนเอง

## Idempotency

- Answer PUT เป็น idempotent
- Import checksum ป้องกัน duplicate batch
- Start session unique ต่อ person/window
- Submit/finalize ป้องกัน double processing

## Versioning

- Breaking change สร้าง /api/v2 หรือ transition plan
- Additive optional field ทำได้เมื่อ client เก่ารับได้
- OpenAPI snapshot เปลี่ยนพร้อม ticket

## Security

- ไม่ส่ง is_correct ก่อน final
- ไม่ส่ง password hash, encrypted identifier หรือ audit secret metadata
- Rate limiting ไม่ใช่ initial infrastructure แต่ endpoint ต้องออกแบบไม่ให้ request เดียวทำงานไร้ขอบเขต
